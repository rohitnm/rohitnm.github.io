Register programming shouldn't require a PDF and a calculator.

Embedded firmware constantly touches hardware registers — GPIO, UART, SPI, DMA config. Every MCU vendor provides SVD files describing their register maps, but working with them is painful.

Reference Manual PDFs

800+ page PDFs. Find the register, find the bit field, find the offset, calculate the mask manually. Context-switch every time.

Vendor HAL Libraries

Abstraction layers that hide the hardware. When you need a specific bit, you're back to reading the register map anyway.

Existing SVD Viewers

Separate tool windows. No cursor awareness — you type GPIOA->ODR and still have to manually search. No code generation.

Manual Bit Math

(1U << 5) — which bit was that again? Off-by-one errors in shift amounts are silent and common.

SVD intelligence, inside your editor.

Cursor-Driven Detection

Place your cursor on GPIOA->ODR or TIM2->CR1 and the HUD automatically shows that register's full bit-field layout.

Bit-Field Visualization

Every field: name, bit range, access type (read/write/read-only), reset value, and description. No more counting bits in hex.

C Code Generation

Click a bit field to generate the correct C code: set, clear, toggle, or read. Correct shift amounts and masks, every time.

Full SVD Tree Browser

Browse the complete peripheral → register → field hierarchy in VS Code's sidebar. Search by name. Jump to any register.

Address & Offset Display

Shows absolute address, base + offset, and reset value for every register. Copy address for debugger watch expressions.

Zero Dependencies

Bundles its own SVD parser. No external tools, no Python, no server. Works offline, works on any OS VS Code supports.

The hard parts of building a register-aware IDE extension.

Cursor-to-Register Matching

C register access patterns are inconsistent: GPIOA->ODR, pGPIOA->ODR, raw address casts. A simple regex catches the common case but misses pointer aliases.

Solution

Multi-strategy matcher: tries PERIPH->REG regex first, then address literal matching against the SVD address map, then #define alias resolution. Covers ~98% of real firmware patterns.

SVD Inheritance & derivedFrom

SVD files use derivedFrom where one peripheral inherits another's register layout (e.g., GPIOB derives from GPIOA). Naive parsing shows empty peripherals.

Solution

Two-pass parse: first builds the base peripheral map, second resolves derivedFrom by cloning the parent's register tree and patching the child's base address. Handles nested chains.

Webview Performance on Large SVDs

An STM32H7 SVD has 80+ peripherals, 2000+ registers, 8000+ bit fields. Rendering the full tree caused 500ms+ lag on every cursor move.

Solution

Lazy rendering: only the active peripheral is sent to the Webview. Sidebar uses VS Code's native virtual scrolling. SVD parsed once at activation, cached in memory. Cursor detection debounced to 50ms.

Bit-Field Overlap Detection

Some vendor SVD files contain overlapping bit fields (bugs in the SVD). Rendering these produces incorrect masks.

Solution

Validation pass after parsing: checks for bit-range overlaps per register and emits VS Code diagnostics (yellow warnings). The HUD highlights overlapping fields in red.

Why this extension exists.

Every time I wrote GPIOA->MODER |= (2U << 10); I had to open the reference manual to double-check the bit offset. Every single time. That's 30 seconds of context-switching, multiplied by hundreds of register accesses per project.

The information already existed in SVD files — machine-readable, vendor-provided. No one had built a VS Code extension that made it cursor-aware and actionable. So I did.

Where it goes from here.

Live Debug Integration

Read register values over SWD/JTAG during debug sessions. Show live hardware state alongside the SVD description.

Multi-Field Code Gen

Select multiple bit fields and generate a single combined write. Useful for peripheral init with 5+ fields.

Register Diff

Compare two register values (expected vs. actual) and highlight which bits differ. For debugging peripheral init.

SVD Auto-Detection

Detect the target MCU from .svd files in the workspace or launch.json. No manual selection needed.
